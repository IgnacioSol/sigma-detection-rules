# T1021.001 — Remote Services: Remote Desktop Protocol

## What this rule detects

Successful logon events via RDP (LogonType 10) on Windows systems. In the context of an attack chain, RDP logons using stolen credentials are the primary mechanism for lateral movement between systems in a network.

## Why this technique matters in SOC

RDP lateral movement is one of the most common techniques observed in ransomware attacks and advanced intrusions. After obtaining credentials through techniques like LSASS dumping (T1003.001), attackers use those credentials to connect via RDP to other machines — spreading through the network without installing any new tools. Because RDP is a legitimate administrative protocol, detection must focus on behavioral context: who is connecting, from where, and at what time.

## How the attack works (high level)

1. Attacker obtains valid credentials (e.g., from LSASS memory dump)
2. Uses the RDP client (`mstsc.exe`) or command-line tools to connect to another machine on port 3389
3. Windows logs a successful logon event (Event ID 4624) with LogonType 10
4. Attacker now has interactive access to the target machine and repeats the process

## Why these specific fields are the detection signal

| Field | Why it matters |
|---|---|
| `EventID: 4624` | An account was successfully logged on — filters for successful authentications only |
| `LogonType: 10` | RemoteInteractive — this specific value maps exclusively to RDP sessions in Windows |

**LogonType 10** is the key discriminator. A logon type of 3 (network) or 2 (interactive) would indicate different access methods. Type 10 means someone opened a full graphical RDP session.

This rule uses the **Windows Security Event Log**, not Sysmon — it captures authentication events at the OS level.

## False positives

- **IT administrators**: RDP is the standard tool for remote server management in most enterprises
- **Helpdesk staff**: remote support sessions to end-user machines
- **Automated management tools**: some monitoring and patch management tools use RDP

**Recommended tuning**: this rule will generate high volume in environments with active RDP usage. Reduce false positives by filtering known admin source IPs, expected logon hours, or by correlating with failed logon events (Event ID 4625) to identify brute-force patterns before the successful logon.

## MITRE ATT&CK

- Tactic: **Lateral Movement** (TA0008)
- Technique: **Remote Services** (T1021)
- Sub-technique: **Remote Desktop Protocol** (T1021.001)
