# T1547.001 — Boot or Logon Autostart Execution: Registry Run Keys

## What this rule detects

Any write operation (`SetValue`) to Windows Registry Run Keys — the mechanism Windows uses to automatically execute programs at logon. Writing to these keys is how attackers establish persistence so their malware survives system reboots.

## Why this technique matters in SOC

Without persistence, an attacker loses access every time the machine reboots. Run Keys are one of the most commonly abused persistence mechanisms because they are simple, reliable, and blend in with legitimate software. A SOC analyst investigating any intrusion should always check Run Keys as part of the persistence analysis phase.

## How the attack works (high level)

1. Attacker gains initial access to a system
2. Adds a registry value under a Run Key pointing to their malware executable
3. Reboots or waits for the user to log off and back on
4. Windows automatically executes the malware on every subsequent logon
5. The attacker maintains access without needing to re-exploit the system

## Why these specific fields are the detection signal

| Field | Why it matters |
|---|---|
| `EventType: SetValue` | Filters for write operations only — read operations on Run Keys are normal and not suspicious |
| `TargetObject contains \CurrentVersion\Run` | Matches both HKCU and HKLM Run Key paths with a single pattern, covering user-level and system-level persistence |
| `TargetObject contains \CurrentVersion\RunOnce` | Covers the RunOnce variant, which executes once then deletes itself — used by some malware installers |

This rule uses **Sysmon Event ID 13** (RegistryValueSet).

## False positives

- **Software installers**: applications like Spotify, Chrome, OneDrive, and Teams add themselves to Run Keys during installation
- **System administrators**: scripts that configure startup applications in automated deployments
- **Windows updates**: some update processes temporarily write to Run Keys

**Recommended tuning**: whitelist known software paths and trusted publisher signatures in a `filter` block.
