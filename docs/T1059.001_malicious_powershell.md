# T1059.001 — Command and Scripting Interpreter: PowerShell

## What this rule detects

PowerShell processes launched with flags associated with malicious use: encoded commands (`-enc`), hidden execution (`-WindowStyle Hidden`), execution policy bypass (`-ExecutionPolicy Bypass`), and no-profile execution (`-NoProfile`). Detecting these flags together significantly narrows false positives compared to monitoring all PowerShell execution.

## Why this technique matters in SOC

PowerShell is a Living off the Land (LotL) tool — it is built into every Windows system, signed by Microsoft, and trusted by most security controls. Attackers use it to download and execute malware in memory (leaving no file on disk), move laterally, and run post-exploitation frameworks like Empire or Cobalt Strike. Because it is a legitimate administrative tool, blocking it entirely is not an option; detection must be based on behavioral patterns.

## How the attack works (high level)

1. Attacker gains initial access to a system
2. Launches PowerShell with flags to hide the window and bypass restrictions
3. Downloads a payload from a remote server and executes it directly in memory using `IEX` (Invoke-Expression) or similar
4. No malicious file is written to disk — the entire attack runs in memory

## Why these specific flags are the detection signal

| Flag | What it does | Why attackers use it |
|---|---|---|
| `-enc` / `-EncodedCommand` | Executes a Base64-encoded command | Obfuscates the actual payload from simple string inspection |
| `-WindowStyle Hidden` | Runs PowerShell with no visible window | User does not see a terminal appear |
| `-ExecutionPolicy Bypass` | Ignores the system policy blocking unsigned scripts | Allows any script to run regardless of restrictions |
| `-NoProfile` | Skips loading user profile on startup | Faster, cleaner execution — common in automated attacks |

This rule uses **Sysmon Event ID 1** (Process Creation), which captures the full command line of every process launched, including all flags passed to PowerShell.

## False positives

- **System administrators**: automated scripts for patch management or configuration often use `-ExecutionPolicy Bypass` and `-NoProfile`
- **Software deployment tools**: SCCM, Ansible, Chocolatey run PowerShell silently in the background
- **Legitimate software installers**: many applications use unsigned PowerShell scripts during installation

**Recommended tuning**: add a `filter` block to exclude known administrative script paths or trusted parent processes.

## MITRE ATT&CK

- Tactic: **Execution** (TA0002)
- Technique: **Command and Scripting Interpreter** (T1059)
