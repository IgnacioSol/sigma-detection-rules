# T1003.001 — OS Credential Dumping: LSASS Memory

## What this rule detects

Any process that opens a handle to `lsass.exe` with memory-read permissions (`0x1010`, `0x1410`, `0x1438`). This is the core action required to dump credentials from LSASS memory.

## Why this technique matters in SOC

LSASS (Local Security Authority Subsystem Service) stores authentication credentials in memory — password hashes, Kerberos tickets, NTLM tokens. If an attacker with admin access can read LSASS memory, they obtain all active credentials on that machine. This is typically one of the first things an attacker does after gaining initial admin access, because it enables lateral movement to other systems using those stolen credentials.

Tools commonly used: Mimikatz, ProcDump, Task Manager (manual dump), custom malware.

## How the attack works (high level)

1. Attacker gains administrator privileges on a Windows machine
2. Opens a process handle to `lsass.exe` requesting read access to its memory
3. Reads the process memory and extracts credential material
4. Uses extracted hashes/tickets for Pass-the-Hash or Pass-the-Ticket attacks against other systems

## Why these specific fields are the detection signal

| Field | Why it matters |
|---|---|
| `TargetImage` ending in `\lsass.exe` | LSASS is the only process that stores credentials — any external read access to it is suspicious |
| `GrantedAccess: 0x1010` | `PROCESS_VM_READ` + `PROCESS_QUERY_LIMITED_INFORMATION` — the exact permissions needed to read process memory |
| `GrantedAccess: 0x1410` | Adds `PROCESS_QUERY_INFORMATION` — used by more advanced dumping tools |
| `GrantedAccess: 0x1438` | Superset of the above — includes additional handle permissions used by Mimikatz |

This is a **Sysmon Event ID 10** (ProcessAccess) event. Sysmon must be deployed and configured to log process access events for this rule to work.

## False positives

- **Antivirus / EDR products** (CrowdStrike, SentinelOne, Windows Defender): security tools legitimately read LSASS memory to detect threats
- **ProcDump.exe** (Microsoft Sysinternals): used by administrators to create crash dumps for debugging
- **LSASS crash dump collection** by Windows Error Reporting (WerFault.exe)

**Recommended tuning**: whitelist known security tool paths in a `filter` block using `condition: selection and not filter`.
